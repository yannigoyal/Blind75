## 📁 Structure

Each problem folder follows this pattern:

```
XX_problem_name/
├── question.md        ← Problem statement, examples, constraints
├── brute_force.cpp    ← Naive approach with TC/SC analysis
├── better.cpp         ← Improved approach (if applicable)
└── optimal.cpp        ← Best solution with full explanation
```

> Not every problem needs all files — some go straight from brute to optimal.
